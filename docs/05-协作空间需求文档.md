# 05 — 协作空间需求文档

> **模块**: 评论讨论 / 文件共享 / 项目日志  
> **关联页面**: `DetailPage` (CollabSpace 子组件)  
> **关联 API**: (Phase 2) `POST /api/projects/{id}/comments`, 文件上传 API  
> **数据表**: `comments`, `files`  

---

## 1. 功能概述

协作空间是 DetailPage 中的多功能协作区，通过 **Tab 切换** 组织三种协作形式：

| Tab | 图标 | 功能 | 状态 |
|-----|------|------|------|
| **💬 讨论** | 评论 | 项目内讨论/反馈/决策记录 | 基础完成 (前端 UI) |
| **📂 文件** | 文件 | 设计稿/文档/资源共享 | 前端 UI 完成 (无真实上传) |
| **📋 项目日志** | 日志 | 活动 feed (动态记录) | 前端 UI 完成 (数据依赖 activities) |

---

## 2. 用户故事

| # | 角色 | 故事 | 验收标准 |
|---|------|------|----------|
| US-01 | 团队成员 | 我想对某个设计给出反馈 | 在讨论 tab 输入评论并发送 |
| US-02 | 团队成员 | 我想查看之前的讨论 | 评论列表按时间倒序显示 |
| US-03 | 团队成员 | 我想共享设计文件 | 文件 tab 上传并显示文件列表 |
| US-04 | 团队成员 | 我想下载别人共享的文件 | 点击文件行的下载箭头 |
| US-05 | 任何人 | 我想了解项目进展过程 | 项目日志 tab 查看 activity feed |
| US-06 | 发起人 | 我想记录重要决策 | (Phase 2) 活动自动/手动记录到 log |

## 3. 数据模型

### 3.1 comments 表

```sql
CREATE TABLE comments (
    id          SERIAL PRIMARY KEY,
    project_id  INT NOT NULL REFERENCES projects(id),
    user_name   VARCHAR(50) NOT NULL,         -- 评论者用户名
    user_color  VARCHAR(7),                  -- 评论者颜色 (冗余存储)
    text        TEXT NOT NULL,                -- 评论内容
    time        TIMESTAMPTZ DEFAULT NOW()     -- 评论时间
);
```

### 3.2 files 表 (FileEntity)

```sql
CREATE TABLE files (
    id          SERIAL PRIMARY KEY,
    project_id  INT NOT NULL REFERENCES projects(id),
    name        VARCHAR(255) NOT NULL,          -- 文件显示名
    icon        VARCHAR(10) DEFAULT '📄',     -- 文件图标 emoji
    uploader    VARCHAR(50),                  -- 上传者用户名
    time        TIMESTAMPTZ DEFAULT NOW(),    -- 上传时间
    file_path   VARCHAR(500),                 -- 实际存储路径 (Phase 2)
    file_size   BIGINT DEFAULT 0               -- 文件大小 (bytes)
);
```

## 5. 字段映射 (重要!)

CollabSpace 接收的数据来自 DetailPage 的数据预处理，存在**字段映射**以兼容前后端不一致：

| 前端字段 | 来源字段映射 | 说明 |
|----------|-------------|------|
| `c.user` | `c.authorName \|\| c.author` | 评论者名字 |
| `c.text` | `c.content \|\| c.text` | 评论内容 |
| `c.time` | `c.time \|\| c.createdAt` | 评论时间 |
| `c.color` | `c.authorColor \|\| c.color` | 评论者颜色 |
| `f.name` | `f.fileName \|\| f.name` | 文件名 |
| `f.uploader` | `f.uploaderName \|\| f.uploader` | 上传者 |
| `f.time` | `f.time \|\| f.createdAt` | 上传时间 |

**预处理代码** (DetailPage.jsx lines 184-195):
```js
p.comments = (detail.comments || []).map(c => ({
  user: c.authorName || c.author || '匿名',
  text: c.content || c.text || '',
  time: c.time || c.createdAt || '',
  color: c.authorColor || c.color || '#999',
}))

p.files = (detail.files || []).map(f => ({
  name: f.fileName || f.name || '文件',
  icon: '📄',
  uploader: f.uploaderName || f.uploader || '未知',
  time: f.time || f.createdAt || '',
}))
```

## 6. API 接口 (Phase 2 - TODO)

### 6.1 发送评论

```
POST /api/projects/{projectId}/comments
Authorization: Bearer {token}
Content-Type: application/json

Request:
{
  "content": "string"   // 评论内容 (必填, 非空)
}

Response (200):
{
  "code": 200,
  "message": "评论发布成功",
  "data": {
    "id": 42,
    "userName": "CurrentUser",
    "userColor": "#D4213D",
    "text": "...",
    "time": "2026-04-24T00:52:00Z"
  }
}
```

### 6.2 上传文件

```
POST /api/projects/{projectId}/files
Authorization: Bearer {token}
Content-Type: multipart/form-data

Request: FormData
  - file: File object
  - fileName: string (可选, 默认取 file.name)

Response (200):
{
  "code": 200,
  "message": "文件上传成功",
  "data": {
    "id": 10,
    "name": "design-v3.fig",
    "icon": "🎨",
    "uploader": "CurrentUser",
    "time": "2026-04-24T00:52:00Z",
    "filePath": "/uploads/pj3/design-v3.fig"
  }
}
```

### 6.3 下载文件

```
GET /api/projects/{projectId}/files/{fileId}
Authorization: Bearer {token}

Response: 文件二进制流 (Content-Disposition: attachment)
```

### 6.4 获取活动日志

活动日志作为 `getProjectDetail` 响应的一部分返回（`activities` 字段），无需独立接口。后端可通过查询 comments/files/tasks 的变更记录聚合生成。
