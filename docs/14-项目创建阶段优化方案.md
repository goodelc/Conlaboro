# 14 — 项目创建阶段优化方案

> **文档目的**: 优化项目创建和查看体验，为不同角色提供不同视图  
> **关联文档**: `13-想法孵化优化方案.md`, `02-项目系统需求文档.md`, `12-项目设计哲学文档.md`  
> **适用范围**: CreatePage、DetailPage、项目视图设计  

---

## 一、核心洞察

### 1.1 用户角色分离

| 用户类型 | 特点 | 他们关心什么 |
|---------|------|-------------|
| **想法提供者** | 有想法，不懂技术/项目管理 | 我的想法变成什么样了？什么时候能用？ |
| **项目参与者** | 有技能，懂协作 | 任务怎么分配？进度如何？我该做什么？ |

### 1.2 关键结论

**普通用户（想法提供者）不需要懂「项目」！**

他们只需要知道：
- ✅ 我的想法有人在做吗？
- ✅ 现在做到哪一步了？
- ✅ 什么时候能用上？

他们**不需要**知道：
- ❌ 什么是里程碑
- ❌ 什么是看板
- ❌ 角色怎么配置
- ❌ 任务怎么拆分

---

## 二、设计方案总览

### 2.1 核心改变

| 原设计 | 新设计 |
|--------|--------|
| 所有人看同样的页面 | 不同角色看不同视图 |
| 发起时要求填写专业信息 | 发起时只填基本信息，其他让AI或队友补充 |
| 必须理解里程碑、看板 | 普通用户只需要看「进度条」和「动态」 |
| 项目管理思维 | 想法追踪思维 |

### 2.2 双视图设计

| 视图 | 适用人群 | 核心内容 |
|------|----------|----------|
| 简单视图 | 想法提供者、普通用户 | 进度条、最新动态、团队信息 |
| 详细视图 | 项目参与者、有经验的用户 | 任务看板、里程碑、角色配置 |

---

## 三、发起阶段优化

### 3.1 简化发起流程

**当前问题**：CreatePage要求填写太多专业信息（角色配置、里程碑规划等）

**优化方案**：想法提供者只需要填最基本的信息

```
┌─────────────────────────────────────────────────────────────┐
│  💡 发起你的想法                                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  你的想法叫什么？                                            │
│  [银发餐桌                              ]                   │
│                                                             │
│  用一句话描述它：                                            │
│  [帮助独居老人轻松订餐的暖心App          ]                   │
│                                                             │
│  详细说说你的想法：                                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 我想做一个帮助独居老人订餐的App...                    │   │
│  │ （越详细越好，这样别人更容易理解并加入）              │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  🤖 其他信息可以让AI帮你生成，或者让队友来补充               │
│                                                             │
│  [🤖 让AI帮我完善]              [直接发布，让队友来规划]    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 三种发起路径

| 路径 | 说明 | 适用场景 |
|------|------|----------|
| AI完善 | AI自动生成角色配置、阶段规划 | 不确定怎么规划的人 |
| 直接发布 | 只发布基本信息，其他留空 | 相信队友会补充 |
| 手动规划 | 自己填写所有信息（原有流程） | 已经想清楚的人 |

### 4.3 信息层级

| 层级 | 内容 | 说明 |
|------|------|------|
| 第一层 | 状态 + 进度条 | 一眼看到当前状态 |
| 第二层 | 最近动态 | 了解正在发生什么 |
| 第三层 | 下一步计划 | 知道接下来做什么 |
| 第四层 | 参与的人 | 看到谁在做 |

### 4.4 交互设计

| 按钮 | 功能 |
|------|------|
| 💬 我有建议 | 打开评论框，提出建议 |
| 🔄 我想参与更多 | 申请加入项目 |
| 📤 分享给朋友 | 分享项目链接 |

### 5.3 与原DetailPage的差异

| 原设计 | 新设计 |
|--------|--------|
| 信息密集，一屏显示所有内容 | 分区清晰，按需展开 |
| 术语专业（里程碑、看板） | 术语友好（阶段规划、任务进度） |
| 无视图切换 | 支持简单/详细视图切换 |

## 六、视图切换机制

### 6.1 切换入口

```
┌─────────────────────────────────────────────────────────────┐
│  📱 银发餐桌                                                │
│                                                             │
│  [📋 简单视图]  [🔧 详细视图]                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 默认视图规则

| 用户身份 | 默认视图 | 说明 |
|----------|----------|------|
| 想法发起人 | 简单视图 | 发起人通常关心进展，不需要管理细节 |
| 项目参与者 | 详细视图 | 参与者需要看任务、认领工作 |
| 访客（未登录） | 简单视图 | 先给简单信息，感兴趣再深入 |
| 已登录但未参与 | 简单视图 | 降低理解门槛 |

### 6.3 记忆用户偏好

- 用户手动切换视图后，记住选择
- 下次访问同一项目时，恢复上次选择的视图
- 不同项目可以有不同的视图偏好

## 八、实现要点

### 8.1 数据复用

简单视图和详细视图使用**相同的数据源**，只是展示方式不同：

| 数据 | 简单视图展示 | 详细视图展示 |
|------|-------------|-------------|
| 任务列表 | 计算进度百分比 | 显示任务看板 |
| 里程碑 | 提取下一步计划 | 显示完整时间线 |
| 团队成员 | 显示头像列表 | 显示角色分配 |
| 活动记录 | 显示最近3条 | 显示完整动态 |

### 8.2 组件复用

```
DetailPage
├── ViewSwitcher（视图切换器）
├── SimpleView（简单视图）
│   ├── ProgressSection（进度区）
│   ├── RecentActivity（最近动态）
│   ├── NextSteps（下一步计划）
│   └── TeamAvatars（团队头像）
├── DetailedView（详细视图）
│   ├── TaskBoard（任务看板）
│   ├── MilestoneTimeline（里程碑时间线）
│   ├── TeamRoles（团队角色）
│   └── CollabSpace（协作空间）
└── CommonSections（公共区域）
    ├── ProjectHeader（项目头部）
    └── ActionButtons（操作按钮）
```
