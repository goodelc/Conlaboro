# 15 — 协作阶段优化方案

> **文档目的**: 优化项目协作体验，让不同水平的用户都能参与并有所收获  
> **关联文档**: `14-项目创建阶段优化方案.md`, `05-协作空间需求文档.md`, `04-任务看板需求文档.md`  
> **适用范围**: 任务系统、协作空间、成果展示  

---

## 一、核心设计理念

### 1.1 协作阶段的目标

| 目标 | 说明 |
|------|------|
| 让普通人也能看懂 | 不需要懂项目管理术语 |
| 让有技能的人能高效协作 | 提供专业工具 |
| 让成就感可见 | 看到自己的贡献 |
| 让协作过程简单愉快 | 降低沟通成本 |

### 1.2 双视图延伸

协作阶段延续「简单视图」和「详细视图」的设计：

| 视图 | 适用人群 | 核心内容 |
|------|----------|----------|
| 简单视图 | 想法提供者、普通用户 | 整体进度、谁在做什么、最近完成了什么 |
| 详细视图 | 项目参与者、有经验的用户 | 完整任务看板、里程碑、详细协作讨论 |

## 二、任务难度分级

### 2.1 三级难度体系

```
┌─────────────────────────────────────────────────────┐
│  任务难度                                            │
│                                                      │
│  🌱 入门级 - 新手友好                               │
│     不需要经验，有人带你做                           │
│     预计时间：1-3小时                               │
│                                                      │
│  🌿 进阶级 - 需要一些基础                           │
│     有相关经验会更顺手                               │
│     预计时间：3-8小时                               │
│                                                      │
│  🌳 高级 - 核心功能                                 │
│     需要较深的专业知识                               │
│     预计时间：8小时以上                             │
└─────────────────────────────────────────────────────┘
```

### 2.2 难度价值对照

| 难度 | 适合人群 | 价值 | 预计时间 |
|------|----------|------|----------|
| 🌱 入门级 | 新人、学生、想学习的人 | 降低参与门槛，让新人能快速融入 | 1-3小时 |
| 🌿 进阶级 | 有一定经验的人 | 发挥技能，积累项目经验 | 3-8小时 |
| 🌳 高级 | 资深开发者 | 挑战核心问题，展示能力 | 8小时以上 |

### 2.3 技能标签系统

每个任务可以打上技能标签，帮助匹配：

```
┌─────────────────────────────────────────────────────┐
│  登录功能开发                                        │
│  难度：🌿 进阶级                                     │
│  技能：[前端] [React] [表单处理]                     │
│  预计：3-5小时                                       │
│                                                      │
│  [我想做这个任务！]                                  │
└─────────────────────────────────────────────────────┘
```

### 2.4 用户技能匹配

用户注册时选择技能和经验水平：

```
┌─────────────────────────────────────────────────────┐
│  你会哪些技能？（可多选）                            │
│                                                      │
│  [✓] 前端开发    [ ] 后端开发                       │
│  [ ] 设计        [✓] 产品                           │
│  [ ] 测试        [ ] 运维                           │
│                                                      │
│  你的经验水平？                                      │
│  ○ 新手（想学习）                                    │
│  ● 有一些经验                                        │
│  ○ 资深                                             │
└─────────────────────────────────────────────────────┘
```

系统智能推荐：
- 新手 → 推荐入门级任务
- 有经验 → 推荐进阶级任务
- 资深 → 推荐高级任务

### 2.5 任务展示优化

用大白话描述任务，让每个人都能理解：

```
┌─────────────────────────────────────────────────────┐
│  这阶段我们要做这些事情                              │
│                                                      │
│  [✅ 已完成] 调研老人需求                            │
│  [✅ 已完成] 设计界面原型                            │
│  [🔄 进行中] 登录功能开发  → 小明正在做              │
│  [🔄 进行中] 首页开发  → 小红正在做                  │
│  [💡 可加入] 订餐功能开发  → 谁想加入？              │
│  [💡 可加入] 支付功能开发  → 谁想加入？              │
│                                                      │
│  [我想做订餐功能开发]                                │
└─────────────────────────────────────────────────────┘
```

**优化点**：
- 用「我想做」替代「认领」
- 任务描述用大白话，不用技术术语
- 显示谁在做什么，增加透明感

## 三、协作节奏

### 3.1 项目自选节奏

项目发起时选择节奏，不强制统一：

```
┌─────────────────────────────────────────────────────┐
│  项目节奏                                            │
│                                                      │
│  ○ 🔥 热火朝天                                       │
│     每天都有进展，适合时间充裕的团队                 │
│     建议：每日简单同步                               │
│                                                      │
│  ● 🌊 稳步推进                                       │
│     每周有进展，适合业余时间参与                     │
│     建议：每周一次同步                               │
│                                                      │
│  ○ 🐢 慢慢来                                         │
│     有空就做，没有时间压力                           │
│     建议：有进展时再说                               │
│                                                      │
│  ○ ❓ 不设节奏                                       │
│     完全随意，大家看着办                             │
└─────────────────────────────────────────────────────┘
```

### 3.2 不强制统一的原因

| 原因 | 说明 |
|------|------|
| 为爱发电 | 大家都是自愿参与，强制节奏会让人有压力 |
| 时间不同 | 有人全职，有人兼职，有人只有周末 |
| 项目不同 | 小项目可以快，大项目需要慢 |
| 避免烂尾 | 强制节奏可能导致压力过大而放弃 |

### 3.3 不同节奏的同步方式

| 节奏 | 同步方式 | 工具 |
|------|----------|------|
| 🔥 热火朝天 | 每日站会（5分钟） | 自动提醒 + 简单打卡 |
| 🌊 稳步推进 | 每周总结 | 自动生成周报 |
| 🐢 慢慢来 | 有进展就更新 | 完成任务时自动通知 |
| ❓ 不设节奏 | 无 | 无 |

### 3.4 自动同步工具

系统自动生成同步内容：

```
┌─────────────────────────────────────────────────────┐
│  📋 本周进展（自动生成）                             │
│                                                      │
│  ✅ 完成了 3 个任务                                  │
│  🔄 进行中 2 个任务                                  │
│  💡 新增 1 个任务                                    │
│                                                      │
│  本周亮点：                                          │
│  • 小明完成了登录功能                                │
│  • 小红设计了订单页面                                │
│                                                      │
│  下周计划：                                          │
│  • 完成订餐核心功能                                  │
│  • 开始支付功能开发                                  │
└─────────────────────────────────────────────────────┘
```

## 五、沟通结构化

### 5.1 沟通分区设计

避免信息淹没，将沟通分类：

```
┌─────────────────────────────────────────────────────┐
│  🔔 重要决策 (2)                                    │
│  ├─ 我们用 React，不是 Vue → 已记录                 │
│  └─ 设计稿用 Figma → 已记录                         │
│                                                      │
│  📋 待确认 (1)                                      │
│  └─ 配色方案用哪一个？                              │
│                                                      │
│  💬 自由讨论 (8)                                    │
│  ├─ 这个功能很好！                                  │
│  └─ 同意！我们就这样做。                            │
└─────────────────────────────────────────────────────┘
```

### 5.2 沟通分区说明

| 分区 | 用途 | 特点 |
|------|------|------|
| 🔔 重要决策 | 记录关键决策 | 不会被淹没，方便回溯 |
| 📋 待确认 | 需要讨论决定的问题 | 突出显示，提醒关注 |
| 💬 自由讨论 | 日常交流 | 轻松自由 |

## 七、总结

### 7.1 核心优化点

| 优化项 | 说明 | 核心理念 |
|--------|------|----------|
| 任务难度分级 | 三级难度 + 技能标签 | 让新人能入门，让高手有挑战 |
| 协作节奏 | 项目自选，不强制 | 尊重每个人的时间，避免压力 |
| 成果分享 | 小步展示 + 最终汇总 | 即时反馈 + 长期价值 |
| 沟通结构化 | 决策/待确认/讨论分开 | 重要信息不淹没 |
| 新手引导 | 首次参与有指引 | 降低参与门槛 |

### 7.2 与设计哲学的对应

| 设计哲学 | 协作阶段体现 |
|----------|-------------|
| 降低参与门槛 | 任务难度分级、新手引导 |
| 透明化协作 | 谁在做什么、进度可视化 |
| 正向激励 | 成果展示、即时反馈 |
| 信任构建 | 贡献记录、技能认证 |
