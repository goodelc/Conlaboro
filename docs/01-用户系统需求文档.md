# 01 — 用户系统需求文档

> **模块**: 用户注册 / 登录 / 认证 / 个人资料  
> **关联页面**: `LoginPage`, `RegisterPage`, `ProfilePage`, `SettingsPage`(部分)  
> **关联 API**: `POST /api/auth/register`, `POST /api/auth/login`, `GET /api/auth/me`, `GET /api/users/{id}`  
> **数据表**: `users`  

---

## 1. 功能概述

用户系统是 Conlaboro 的身份基础层，负责：
- 新用户注册（邮箱+密码+昵称+角色标签）
- 已用户登录（JWT Token 认证）
- 当前用户会话保持（localStorage 持久化 + Axios 自动附加 Token）
- 个人主页公开展示（ProfilePage）
- 个人资料管理（SettingsPage）

---

## 2. 用户故事 (User Stories)

| # | 角色 | 故事 | 验收标准 |
|---|------|------|----------|
| US-01 | 访客 | 我想注册一个账号来参与项目 | 填写昵称/邮箱/密码后成功创建账号，自动登录并跳转 Dashboard |
| US-02 | 访客 | 我已有账号，想登录 | 输入邮箱+密码后获取 JWT，跳转 Dashboard |
| US-03 | 注册用户 | 我忘记密码了 | （Phase 3 — TODO: 找回密码流程） |
| US-04 | 任何用户 | 我想查看某个人的公开资料 | 访问 `/profile/:name` 看到其 XP/项目/徽章/技能 |
| US-05 | 登录用户 | 我想修改我的昵称/简介/技能 | SettingsPage 中点击「修改」→ 弹窗编辑 → 保存到 localStorage |

## 3. 数据模型

### 3.1 users 表

```sql
CREATE TABLE users (
    id            SERIAL PRIMARY KEY,
    name          VARCHAR(50) NOT NULL UNIQUE,       -- 昵称（唯一标识）
    email         VARCHAR(100) NOT NULL UNIQUE,      -- 邮箱
    password_hash VARCHAR(255) NOT NULL,              -- BCrypt 哈希
    color         VARCHAR(7) DEFAULT '#6366f1',      -- 头像背景色 hex
    role          VARCHAR(50) DEFAULT '新社员',       -- 主角色标签
    level         INT DEFAULT 1,                     -- 等级 1-7
    level_name    VARCHAR(20) DEFAULT '新社员',       -- 等级名称
    xp            INT DEFAULT 0,                      -- 贡献值
    project_count INT DEFAULT 0,                     -- 参与项目数
    badge_count   INT DEFAULT 0,                     -- 徽章数
    joined_at     TIMESTAMPTZ DEFAULT NOW(),          -- 加入时间
    bio           TEXT,                              -- 个人简介
    deleted       INTEGER DEFAULT 0                 -- 逻辑删除 (@TableLogic)
);
```

### 3.2 关联表

| 表名 | 用途 | 关联字段 |
|------|------|----------|
| `user_badges` | 用户已获得徽章 | `userId → users.id` |
| `user_skills` | 用户技能及熟练度 | `userId → users.id` |
| `notifications` | 通知 | `userId → users.id` |
| `projects` | 发起的项目 | `authorId → users.id` |
| `project_roles` | 项目角色成员 | 通过中间关系 |

## 4. 页面规格

### 4.1 LoginPage `/login`

**布局**: 全屏居中 `.auth-card`，max-width: **440px**

**结构**:
```
┌─────────────────────────────┐
│  ⭐ 共创公社                  │  ← auth-logo (Noto Serif SC 900)
│  登录你的创客身份             │  ← auth-subtitle (warm-gray)
│                             │
│  [ GitHub ]  OAuth 按钮     │  ← .auth-oauth-btn (TODO: 未实现)
│  [ Google ] OAuth 按钮      │
│  ─── 或 ───                 │  ← .auth-divider
│  ┌─────────────────────┐   │
│  │ 📧 邮箱             │   │  ← form-input
│  └─────────────────────┘   │
│  ┌─────────────────────┐   │
│  │ 🔒 密码             │   │
│  └─────────────────────┘   │
│                             │
│  [        登 录         ]   │  ← btn-primary (width: 100%)
│                             │
│  还没有账号？[立即注册 →]    │  ← auth-switch
└─────────────────────────────┘
```

**交互逻辑**:
1. 校验：邮箱和密码非空
2. 调用 `login(email, password)` from AppContext
3. 内部调用 `POST /api/auth/login`
4. 成功 → `dispatch({ type: 'LOGIN', user: res })` → 写入 localStorage → `navigate('/dashboard')`
5. 失败 → `showToast(err.message, 'error')`

**前端文件**: `frontend/src/pages/LoginPage.jsx`

### 4.2 RegisterPage `/register`

**布局**: 同 LoginPage 结构，但 card 更宽 max-width: **480px**，多一个「角色选择」区块

**表单字段**:

| 字段 | 类型 | 必填 | 验证规则 | 默认值 |
|------|------|------|----------|--------|
| 昵称 | text | ✅ | 非空 | placeholder: `你的公社昵称` |
| 邮箱 | email | ✅ | 非空 | placeholder: `your@email.com` |
| 密码 | password | ✅ | ≥ 6 位 | placeholder: `至少6位` |
| 角色标签 | multi-select chip | ❌ | 5 选多 | 🎯PM / 🎨设计 / 💻开发 / 🧪测试 / 📣运营 |

**角色选择 UI**:
- 使用 `.role-tags` 容器，每个选项为 `<span class="role-tag">`
- 点击 toggle：`.role-tag.active` = 红底白字
- 存储在 `form.roles: string[]`

**交互逻辑**:
1. 校验必填项 + 密码长度 ≥ 6
2. 调用 `register(name, email, password)` from AppContext
3. 内部调用 `POST /api/auth/register`
4. 成功 → 自动 LOGIN（注册即登录）→ `navigate('/dashboard')`
5. showToast 欢迎消息

**前端文件**: `frontend/src/pages/RegisterPage.jsx`

## 5. API 接口

### 5.1 注册

```
POST /api/auth/register
Content-Type: application/json

Request:
{
  "name": "string",      // 昵称 (必填)
  "email": "string",     // 邮箱 (必填)
  "password": "string"   // 密码 (必填, ≥6位)
}

Response (200):
{
  "code": 200,
  "message": "success",
  "data": {
    "userId": 1,
    "name": "...",
    "email": "...",
    "token": "jwt-string",
    "color": "#6366f1",
    "role": "member",
    "level": 1,
    "levelName": "Newcomer",
    "xp": 0,
    "projectCount": 0,
    "badgeCount": 0
  }
}
```

### 5.2 登录

```
POST /api/auth/login
Content-Type: application/json

Request:
{
  "email": "string",
  "password": "string"
}

Response: 同 register 返回结构（含 token）
```

### 5.3 获取当前用户

```
GET /api/auth/me
Headers: Authorization: Bearer {token}

Response (200):
{
  "code": 200,
  "data": { /* User 对象 */ }
}
```

### 5.4 获取用户详情

```
GET /api/users/{id}
Headers: Authorization: Bearer {token}

Response (200):
{
  "code": 200,
  "data": {
    "id", "name", "email", "color", "role",
    "level", "levelName", "xp", "projectCount",
    "badgeCount", "joinedAt", "bio",
    "skills": [{ "name": "React", "percentage": 90 }],
    "earnedBadges": [0, 1, 3]
  }
}
```

## 6. 认证机制

### 6.1 JWT 流程

```
[客户端]                     [服务端]
   │                             │
   ├── POST /auth/login ──────→  │
   │                             │ 验证邮箱密码
   │                             │ 生成 JWT (JJWT)
   │ ←── { token, user } ──────  │
   │                             │
   │  存储 localStorage:         │
   │  { isLoggedIn, currentUser } │
   │                             │
   ├── GET /api/projects ─────→  │
   │  Header: Bearer {token}    │  JwtAuthFilter 验证
   │                             │ setAttribute("userId")
   │ ←── 项目列表 ────────────  │
```

### 6.2 前端 Axios 拦截器 (`request.js`)

**请求拦截器**:
```js
// 从 localStorage 读取 token
const saved = JSON.parse(localStorage.getItem('conlaboro_auth'))
if (saved?.currentUser?.token) {
  config.headers.Authorization = `Bearer ${saved.currentUser.token}`
}
```

**响应拦截器**:
```js
// 解包 data 字段: return res.data
// code !== 200 → 抛错
// 401 → 清除登录态 → 跳转 /login
// HTTP 401 → 同上
```

### 6.3 AppContext 状态

**LOGIN action** 提取的字段映射:

```js
const authUser = {
  id: user.userId,
  name: user.name,
  email: user.email,
  token: user.token,
  color: user.color || '#6366f1',
  role: user.role || 'member',
  level: user.level || 1,
  levelName: user.levelName || 'Newcomer',
  xp: user.xp || 0,
  projects: user.projectCount || 0,
  badges: user.badgeCount || 0,
}
```

**持久化 Key**: `conlaboro_auth`
```json
{ "isLoggedIn": true, "currentUser": { ...authUser } }
```
