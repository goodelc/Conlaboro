# 03 — 团队与角色需求文档

> **模块**: 团队组建 / 角色配置 / 申请加入 / 成员管理  
> **关联页面**: `CreatePage`(步骤②), `DetailPage`(团队成员+侧边栏招募)  
> **关联组件**: `JoinModal`, `ProjectCard`(角色 badges)  
> **关联 API**: (Phase 2) `POST /api/projects/{id}/apply`  
> **数据表**: `project_roles`, 中间关系表  

---

## 1. 功能概述

团队与角色系统实现 Conlaboro「自愿加入」「透明协作」的核心协作理念。每个项目由发起人定义需要的角色和人数，其他用户可以申请加入。

**核心流程**:
```
发起人创建项目 → 定义角色+人数 → 公开招募
                        ↓
              用户浏览项目 → 申请加入特定角色 → (审核) → 成为团队成员
```

---

## 2. 用户故事

| # | 角色 | 故事 | 验收标准 |
|---|------|------|----------|
| US-01 | 发起人 | 我想定义项目需要哪些角色 | CreatePage 步骤② 配置 5 种角色的需求数量 (0~5人) |
| US-02 | 发起人 | 我想限制某个角色不需要 | 将该角色数量设为 0 |
| US-03 | 访客 | 我想查看项目还缺什么角色 | DetailPage 侧边栏「角色招募」区域看到各角色缺口 |
| US-04 | 登录用户 | 我想加入某个项目 | 点击「申请」或「加入项目」→ 打开 JoinModal |
| US-05 | 登录用户 | 我想以特定角色加入 | JoinModal 中选择目标角色 + 填写自我介绍 |
| US-06 | 任何人 | 我想看谁已经在项目中 | DetailPage「团队成员」区域看到已入驻成员 |

## 3. 数据模型

### 3.1 project_roles 表

```sql
CREATE TABLE project_roles (
    id        SERIAL PRIMARY KEY,
    project_id INT NOT NULL REFERENCES projects(id),
    name      VARCHAR(50) NOT NULL,    -- 角色显示名
    emoji     VARCHAR(10),             -- 角色 emoji 图标
    needed    INT NOT NULL DEFAULT 1,  -- 需要的总人数
    filled    INT NOT NULL DEFAULT 0   -- 已填充人数
);
```

### 3.2 角色预置体系

CreatePage 中硬编码的 5 种基础角色：

| # | Emoji | 名称 | 默认需求数 | 适用场景 |
|---|-------|------|-----------|----------|
| 1 | 🎯 | 产品经理 | 1 | 所有项目通常需要 |
| 2 | 🎨 | 设计师 | 1 | UI/UX/视觉设计 |
| 3 | 💻 | 前端开发 | 1 | React/Vue 等 |
| 4 | ⚙️ | 后端开发 | 1 | API/数据库/架构 |
| 5 | 🧪 | 测试工程师 | 0 | 可选 |

**扩展性**: 项目实际运行中可能超出这 5 种（如游戏策划🎮、像素美术🎨、音效🔊等），种子数据中已有自定义角色案例。

### 3.3 成员关系

当前设计中，`project_roles.filled` 字段记录已填充人数。详细成员信息通过以下方式关联：

**方式 A (种子数据)**: role 对象内嵌 `members` 数组
```json
{
  "name": "产品经理",
  "emoji": "🎯",
  "needed": 1,
  "filled": 1,
  "members": [
    { "name": "Lina", "color": "#D4213D" }
  ]
}
```

**方式 B (API)**: 后端通过中间表（如 `project_members`）查询成员详情

### 4.2 DetailPage — 角色招募侧边栏 (`.sidebar-card`)

**位置**: `detail-sidebar` 第二个 card

**结构**:
```
┌─ 角色招募 ─────────────────────────┐
│                                    │
│  🎯 产品经理        [申请]          │  ← open (红色边框)
│  🎨 设计师          [申请]          │  ← open
│  💻 前端开发        [申请]          │  ← open
│  ⚙️ 后端开发        已满            │  ← filled (cream背景)
│                                    │
└────────────────────────────────────┘
```

**渲染逻辑**:
```js
{p.roles.map(r => {
  const openSlots = r.needed - r.filled
  return openSlots > 0 ? (
    <div className="role-slot open">
      <div className="slot-left"><span>{r.emoji}</span><span>{r.name}</span></div>
      <button className="slot-btn" onClick={() => openJoinModal(p.id, r.name)}>
        申请
      </button>
    </div>
  ) : (
    <div className="role-slot filled">
      <div className="slot-left"><span>{r.emoji}</span><span>{r.name}</span></div>
      <span style={{ color: 'var(--success)' }}>已满</span>
    </div>
  )
})}
```

**CSS 类**:
- `.role-slots`: flex column, gap 0.6rem

- `.role-slot.open`: border-color red, cursor pointer, hover bg red-5%

### 4.4 JoinModal (全局弹窗)

**触发入口** (3处):
1. DetailPage hero 操作区 → 「🤝 加入项目」按钮
2. DetailPage sidebar 角色招募 → 每个 open 角色的「申请」按钮
3. DetailPage team-list 空缺位 → 点击 `?` 卡片

**弹窗结构**:
```
┌─ modal-overlay ──────────────────────────────┐
│                                               │
│  ┌─ modal ────────────────────────────────┐   │
│  │                                       │   │
│  │  🤝 申请加入项目                      │   │
│  │  申请加入「{project.title}」           │   │
│  │                                       │   │
│  │  选择角色 *                            │   │
│  │  ┌─────────────────────────────┐     │   │
│  │  │ ▼ [🎯 产品经理          ▼]  │     │   │  ← select
│  │  └─────────────────────────────┘     │   │
│  │                                       │   │
│  │  自我介绍 *                            │   │
│  │  ┌─────────────────────────────┐     │   │
│  │  │ 简单介绍你自己，为什么想    │     │   │  ← textarea
│  │  │ 加入这个项目...             │     │   │
│  │  │                             │     │   │
│  │  └─────────────────────────────┘     │   │
│  │                                       │   │
│  │     [取消]        [提交申请]       │   │
│  └───────────────────────────────────────┘   │
│                                               │
└───────────────────────────────────────────────┘
```

**交互逻辑**:
```js
// 打开时自动选中第一个可用角色或 preselectedRole
useEffect(() => {
  if (joinModalOpen && openRoles.length > 0) {
    setSelectedRole(preselectedRole || openRoles[0].name)
  }
  setIntro('')
}, [joinModalOpen, preselectedRole, openRoles])

// 提交校验
function handleSubmit() {
  if (!intro.trim()) { showToast('请填写自我介绍', 'info'); return }
  showToast(`已申请「${selectedRole}」角色，等待审核`, 'success')
  closeJoinModal()
}
```

**关闭方式**: 点击遮罩层 / ESC 键

## 5. API 接口 (Phase 2 - TODO)

### 5.1 申请加入项目

```
POST /api/projects/{projectId}/apply
Authorization: Bearer {token}
Content-Type: application/json

Request:
{
  "roleId": 1,              // 目标角色 ID (可选, 也可传角色名)
  "roleName": "前端开发",    // 目标角色名称
  "introduction": "我是一名 React 开发者，有 2 年经验..."
}

Response (200):
{
  "code": 200,
  "message": "申请已提交，等待审核",
  "data": null
}
```

### 5.2 查询项目成员 (扩展)

```
GET /api/projects/{projectId}/members
Authorization: Bearer {token}

Response (200):
{
  "code": 200,
  "data": [

      "userName": "Lina",
      "userColor": "#D4213D",
      "roleId": 1,
      "roleName": "产品经理",
      "joinedAt": "2025-04-10T08:00:00Z",
```
