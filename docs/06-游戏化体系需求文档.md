# 06 — 游戏化体系需求文档

> **模块**: 等级系统 / 徽章系统 / XP 贡献值 / 排行榜  
> **关联页面**: `DashboardPage`(荣誉栏+徽章墙+能力图谱), `LeaderboardPage`, `ShowcasePage`(贡献排行), `BadgeModal`  
> **关联组件**: BadgeModal  
> **数据表**: `badges`, `user_badges`, `user_skills`  

---

## 1. 功能概述

游戏化体系是 Conlabro 的差异化核心竞争力，将开源协作转化为 RPG 式的成长体验：

| 子系统 | 作用 | 位置 |
|--------|------|------|
| **等级系统** | 7 级成长路径，反映用户贡献积累 | Dashboard 荣誉栏 |
| **XP 贡献值** | 量化贡献的通用货币 | Dashboard / 排行榜 |
| **徽章系统** | 16 枚成就勋章，记录里程碑 | Dashboard 徽章墙 / BadgeModal |
| **排行榜** | 全局竞争排名 | LeaderboardPage |
| **能力图谱** | 技能可视化 | Dashboard / ProfilePage |

**设计哲学**: 让每一次贡献都被看见、被认可、被纪念。

## 4. XP 贡献值系统

### 4.1 XP 获取途径

| 行为 | XP 范围 | 备注 |
|------|---------|------|
| 注册/完善资料 | +10 ~ +50 | 一次性 |
| 加入项目 | +30 | 首次加入某项目 |
| 提交评论 | +5 | 每条有效评论 |
| 认领任务 | +任务标注 XP | 10~30 不等 |
| 完成任务 | +任务标注 XP | 10~30 不等 |
| 完成里程碑 | +30 | 每个里程碑 |
| 项目上线 | +50 | 发起人或核心成员 |
| 被采纳建议 | +10 | Review 系统 (Phase 3) |
| 连续签到 | +5/天 | 连续 30 天不灭徽章 |

### 4.2 XP 消费 (未来可能的)

| 消费项 | XP 消耗 | 说明 |
|--------|---------|------|
| 发起项目 | - | 免费 (鼓励创造) |
| 特殊头像框 | -500 | 装饰性消费 |
| 置顶公告 | - | 管理员功能 |
| (暂不设计消费体系，XP 单向增长)

## 6. 能力图谱

### 6.1 数据结构

**user_skills 表**:
```sql
CREATE TABLE user_skills (
    id      SERIAL PRIMARY KEY,
    user_id INT NOT NULL REFERENCES users(id),
    name    VARCHAR(50) NOT NULL,        -- 技能名: "React", "Python"...
    percentage INT NOT NULL DEFAULT 0   -- 熟练度 0-100
);
```

**前端格式**:
```javascript
skills: [
  { name: "React", pct: 90 },     // 高级 (≥80)
  { name: "Vue", pct: 60 },       // 中级 (≥50 且 <80)
  { name: "Go", pct: 30 }         // 初级 (<50)
]
```

### 6.2 UI (DashboardPage skills-grid)

**位置**: Dashboard → dash-section (gridColumn: `1 / -1`) → `📊 能力图谱`

**布局**: 3 列 grid, gap 1rem, padding 1.5rem

**技能卡片** (`.skill-item`):
```
┌────────────────────┐
│       ⚛️            │  ← skill-emoji, fs 1.5rem, mb 0.5rem
│      React          │  ← skill-name, fw 700, fs 0.75rem
│      高级           │  ← skill-level, uppercase, fs 0.6rem, color warm-gray
│  ██████████░ 90%   │  ← skill-bar (3px height), skill-bar-fill (red, width=pct%)
└────────────────────────┘
```

- hover: border gold
- 进度条过渡: `transition width 1s ease`

**等级判定**:
