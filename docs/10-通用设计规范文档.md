## 1. 设计系统 (Design System) 速查

### 1.1 色彩体系 (CSS Variables)

```css
:root {
  /* 主色 */
  --red:        #D4213D;    /* 品牌主色 - CTA、强调、链接 hover */
  --red-dark:   #A11A30;    /* 主色深态 */
  --gold:       #D4A843;    /* 金色 - 等级、成就、精选 */
  --gold-light: #F0D68A8;    /* 浅金 */

  /* 背景 */
  --cream:      #F5F0E8;    /* 奶油底色 - 卡片交替背景 */
  --paper:      #FAF7F0;    /* 纸白底色 - 页面主背景 */

  /* 文字 */
  --ink:        #1A1A1A;    /* 主文字 */
  --ink-light:  #3D3D3D;    /* 次文字 */
  --warm-gray:  #8B8578;    /* 辅助文字、占位符 */

  /* 边框 & 功能 */
  --border:     #D4CFC5;    /* 边框线 */
  --success:    #228B22;    /* 成功/已完成状态 */
}
```

**色彩语义**:

| 颜色 | 用途 | 情感 |
|------|------|------|
| `--red` | 主操作 (CTA)、错误、重要强调 | 紧急、行动 |
| `--gold` | 成就、等级、高亮 | 荣耀、庆祝 |
| `--cream` | 交替背景、已满位 | 温和、中性 |
| `--paper` | 页面底色 | 干净、透气 |
| `--ink` | 正文 | 易读、专业 |
| `--success` | 完成/正向状态 | 成功、通过 |

### 1.2 字体系统

| 用途 | 字体家族 | 字重 | 字号基准 |
|------|----------|------|--------|
| **标题/品牌** | `'Noto Serif SC', serif` | 900 (Black), 700 (Bold) | h1: clamp(3rem,8vw,6.5rem) |
| **正文/UI** | `'Noto Sans SC', sans-serif` | 300(Light), 400(Rg), 500(Med), 700(Bd) | body: 16px (base) |
| **数字/徽章** | `'Noto Serif SC', serif` | 900 | 数字用衬线体增强识别度 |

**引入方式**: Google Fonts (无需本地部署):
```css
@import url('https://fonts.googleapis.com/css2?family=Noto+Serif+SC:wght@400;700;900&family=Noto+Sans+SC:wght@300;400;500;700&display=swap');
```

### 1.5 按钮规范

| 类型 | Class | Bg Color | Text Color | Border | 使用场景 |
|------|-------|----------|------------|--------|----------|
| **Primary** | `.btn-primary` | `var(--red)` | white | none | 主要操作（登录/发布/加入/CTA） |
| **Secondary** | `.btn-secondary` | transparent | `var(--ink)` | `2px solid var(--border)` | 次要操作（取消/收藏/Fork） |
| **White CTA** | `.btn-white` | white | `var(--red)` | none | 深色背景上的 CTA |
| **Nav CTA** | `.nav-cta` | 同 Primary | white | compact | 导航栏注册按钮 |

**Primary 按钮交互**:
- hover: `translateY(-2px)` + `bg: var(--red-dark)` + `box-shadow: 0 8px 24px rgba(212,33,61,0.25)`
- **光泽扫过**: `::before` 伪元素 linear-gradient 白色半透明从左滑到右 (`transition: left 0.5s`)
- transition: `all 0.3s ease`

**全局按钮规则**:
- transition: `all 0.3s`
- font-family: `Noto Sans SC`, sans-serif
- font-weight: 700, letter-spacing: `0.08em`
- cursor: pointer

## 3. 动画系统

### 3.1 入场动画

| 名字 | 触发 | 效果 | 使用场景 |
|------|------|------|----------|
| `fadeInUp` | 页面加载/滚动进入 | 从下往上淡入 + 出现 | Hero 文字/步骤卡片 |
| `fadeInDown` | 页面加载 | 从上往下淡入 | Hero badge |

| `modalIn` | Modal 打开 | 缩放 95% + 上移 | 弹窗 |
| `toastIn` | Toast 出现 | 右滑入 | 全局提示 |
| `toastOut` | Toast 消失 | 下滑出 | 全局提示 |

### 3.2 持续/循环动画

| 名称 | 效果 | 参数 |
|------|------|------|
| `heroGlow` | 背景光晕缓慢旋转移动 | 15s infinite alternate |
| `pulse` | 绿点呼吸缩放 | 2s infinite |
| `confetti` | 粒子飘散上浮消失 | 1s forwards |

### 3.3 滚动触发动画

## 5. 编码约定 (必须遵守)

### 5.1 Null 安全访问模式

**原因**: 后端数据可能缺少字段或返回格式不一致

**规则**: 所有从后端获取的数据对象，访问属性前必须兜底：

```jsx
// ✅ 正确写法
const project = p.project || p
const roles = p.roles || []
const members = roles.flatMap(r => r.members || [])
const skills = u.skills || []

// ❌ 错误写法 (会导致崩溃)
const title = project.title  // 如果 project 为 undefined 则崩溃
```

**需兜底的字段映射** (DetailPage):
```jsx
p.description || p.desc
p.authorName || p.author
c.authorName || c.author
f.uploaderName || f.uploader
c.content || c.text
f.fileName || f.name
```

### 5.2 颜色解析

用户颜色存储为 hex 字符串 (`#D4213D`)，转为 rgb 使用：

```jsx
const rgb = parseInt(color.slice(1), 16)
const r = (rgb >> 16) & 255
const g = (rgb >> 8) & 255
const b = rgb & 255
// 使用: background: `rgb(${r},${g},${b})`
```

### 5.3 导航统一

- 页面内导航: 优先 `useNavigate()` hook
- 外部链接: 使用 `<Link to="...">`
- Toast 调用: `showToast('消息', 'success'/'error'/'info') from `useApp()`
- 常量提取: 状态/颜色/枚举映射放在 `constants.js`

### 5.4 文件命名

| 类型 | 格式 | 示例 |
|------|------|------|
| 页面 | PascalCase + `Page.jsx` | `HomePage.jsx`, `DetailPage.jsx` |
| 组件 | PascalCase + `.jsx` | `ProjectCard.jsx`, `Navbar.jsx` |
| Hook | camelCase + `use*.js` | `useScrollReveal.js` |
| Context | PascalCase + `Context.jsx` | `AppContext.jsx`, `DataContext.jsx` |
| API | camelCase + `.js` | `auth.js`, `project.js` |
| 数据 | camelCase + `.js` | `users.js`, `badges.js` |
| 常量 | camelCase + `.js` | `constants.js` |
| 样式 | kebab-case + `.css` | `styles.css` |
| 测试 | 同文件名 + `.test.{js,jsx}` | `ProjectCard.test.jsx` |

### 5.5 状态管理

- 使用 `useReducer` + Context (不用 Redux/Zustand)
- 全局 UI 状态 → `AppContext` (AppProvider)
- 全局数据状态 → `DataContext` (DataProvider)
- 持久化 Key → `localStorage.getItem('conlaboro_auth')`
- 异步操作 → useCallback 包装避免重复渲染

### 5.6 Modal 规范

所有 Modal 统一遵循:
- 外层: `.modal-overlay.active` (fixed fullscreen semi-transparent black)
- 内层: `.modal` (white card, max-width: 500px, padding: 2.5rem)
- 标题: `font-family: 'Noto Serif SC', serif; font-weight: 900; font-size: 1.3rem`
- 支持点击遮罩关闭 + ESC 键关闭

## 7. 后端分层架构速查

| 层 | 技术 | 职责 |
|----|------|------|
| `controller/` | REST 端点 | 接收请求、参数校验、调用 Service |
| `service/` | 业务逻辑 | 事务编排、业务规则判断 |
| `mapper/` (MyBatis-Plus) | 数据访问 | SQL 构建、CRUD |
| `entity/` | 数据对象 | Lombok @Data 注解类 |
| `dto/` | 传输对象 | 请求/响应 DTO |
| `common/` | `Result<T>` / ErrorCode` | 统一响应封装 |
| `config/` | CORS / Security / JwtFilter | 跨域、认证、过滤器 |
| `security/` | TokenProvider / AuthFilter / UserPrincipal | JWT 生成与验证 |
| `exception/` | GlobalExceptionHandler | 全局异常捕获 |

### 7.1 后端关键约定

| 约定 | 说明 |
|------|------|
| 时间类型 | PostgreSQL 用 `TIMESTAMPTZ` → Java 用 `OffsetDateTime` (**禁止 LocalDateTime!**) |
| 用户标识 | Controller 用 `@RequestAttribute("userId")` 获取 (JwtAuthFilter 中 setAttribute) |
| 统一响应 | `Result<T>{ code, message, data }` 封装 |
| 逻辑删除 | `@TableLogic private Integer deleted` |
| 自增序列 | 手工插入 seed 后需 `setval()` 重置序列 (见 V3 migration) |
| CORS | 开发环境已放开 (`CorsConfig.java`) |
