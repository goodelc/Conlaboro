# 02 — 项目系统需求文档

> **模块**: 项目创建 / 浏览 / 搜索 / 详情  
> **关联页面**: `CreatePage`, `DetailPage`, `HomePage`(项目浏览区), `ShowcasePage`  
> **关联组件**: `ProjectCard`  
> **关联 API**: `GET /api/projects`, `GET /api/projects/{id}`, `POST /api/projects`  
> **数据表**: `projects`, `project_roles`, `milestones`, `tasks`  

---

## 1. 功能概述

项目系统是 Conlaboro 的核心业务模块，覆盖项目完整生命周期：

| 阶段 | 功能 | 页面 |
|------|------|------|
| **发起** | 创建项目（三步表单） | CreatePage `/create` |
| **发现** | 浏览 / 搜索 / 分类筛选 | HomePage `#projects` 区块 |
| **了解** | 项目详情（看板/评论/里程碑/团队） | DetailPage `/detail/:id` |
| **展示** | 已完成项目成果展厅 | ShowcasePage `/showcase` |

## 2. 用户故事

| # | 角色 | 故事 | 验收标准 |
|---|------|------|----------|

## 3. 数据模型

### 3.1 projects 表

```sql
CREATE TABLE projects (
    id              SERIAL PRIMARY KEY,
    title           VARCHAR(200) NOT NULL,         -- 项目名称
    description     TEXT,                          -- 详细描述
    status          VARCHAR(20) NOT NULL DEFAULT 'open',  -- open/progress/done
    category        VARCHAR(20),                   -- app/web/game/tool/ai/other
    author_id       INT REFERENCES users(id),
    author_name     VARCHAR(50),                   -- 冗余存储发起人姓名
    author_color    VARCHAR(7),                    -- 发起人颜色
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    duration        VARCHAR(20),                   -- 预计周期: "2 周"/"1 月"等
    license         VARCHAR(50) DEFAULT 'MIT',     -- 开源协议
    total_hours     INT DEFAULT 0,                 -- 总工时(完成后统计)
    total_commits   INT DEFAULT 0,                 -- 总提交数
    completed_at    TIMESTAMPTZ                    -- 完成时间
    deleted         INTEGER DEFAULT 0             -- @TableLogic
);
```

### 3.2 project_roles 表

```sql
CREATE TABLE project_roles (
    id        SERIAL PRIMARY KEY,
    project_id INT REFERENCES projects(id),
    name      VARCHAR(50) NOT NULL,    -- 角色名: "产品经理"/"设计师"/...
    emoji     VARCHAR(10),             -- 角色图标 emoji
    needed    INT DEFAULT 1,            -- 需要人数
    filled    INT DEFAULT 0             -- 已填人数
);
```

### 3.3 milestones 表

```sql
CREATE TABLE milestones (
    id          SERIAL PRIMARY KEY,
    project_id  INT REFERENCES projects(id),
    title       VARCHAR(200) NOT NULL,
    status      VARCHAR(20) DEFAULT 'pending',  -- done/current/pending
    date        VARCHAR(50),             -- "4月10日 - 4月20日"
    sort_order  INT DEFAULT 0
);
```

### 3.4 tasks 表

```sql
CREATE TABLE tasks (
    id          SERIAL PRIMARY KEY,
    milestone_id INT REFERENCES milestones(id),
    name        VARCHAR(200) NOT NULL,
    status      VARCHAR(20) DEFAULT 'open',    -- open/progress/done
    assignee    VARCHAR(50),                   -- 负责人用户名
    xp          INT DEFAULT 10                -- XP 奖励值
);
```

### 3.5 Wrapper 格式（关键！）

`GET /api/projects` 返回的每个元素是 **wrapper 对象**：

```json
{
  "project": { "id": 1, "title": "RecipeHub", "status": "open", ... },
  "roles": [
    { "id": 1, "name": "产品经理", "emoji": "🎯", "needed": 1, "filled": 1 }
  ]
}
```

> **所有消费组件必须使用安全访问模式**:
> ```js
> const project = p.project || p      // 兼容裸项目和 wrapper
> const roles = p.roles || []         // 兜底空数组
> ```

## 4. 页面规格

### 4.1 CreatePage `/create` (需认证)

**三步表单**, max-width: **800px**

#### 步骤 ① 基本信息 (`.form-section`)

| 字段 | 类型 | 必填 | 选项 |
|------|------|------|------|
| 项目名称 | text | ✅ | placeholder: "给你的项目起个名字" |
| 一句话描述 | text | ✅ | placeholder: "用一句话说明这个项目做什么" |
| 详细描述 | textarea | ✅ | min-height: 120px, placeholder 含引导文案 |
| 项目类别 | select | ❌ | app/web/game/tool/ai/other |
| 预计周期 | select | ❌ | 2周/1月/2月/3月+(默认1月) |

#### 步骤 ② 团队角色配置 (`.role-setup`)

预置 5 种角色，每种可 ± 调整人数（范围 0~5）：

| Emoji | 角色名 | 默认人数 |
|-------|--------|----------|
| 🎯 | 产品经理 | 1 |
| 🎨 | 设计师 | 1 |
| 💻 | 前端开发 | 1 |
| ⚙️ | 后端开发 | 1 |
| 🧪 | 测试工程师 | 0 |

UI: `.role-setup-item` 行内显示 emoji + 名称 + `[−] [count] [+]` 按钮

#### 步骤 ③ 里程碑规划

3 个文本输入框，占位示例：
- 里程碑 1: `例：完成需求文档和原型设计`
- 里程碑 2: `例：完成核心功能开发`
- 里程碑 3: `例：测试、修复、上线`

#### 提交与操作

- 底部 `.form-actions`: [取消] `.btn-secondary` → navigate('/home') | [🚀 发布项目] `.btn-primary`
- 校验: name/tagline/desc 三者非空 → 否则 toast "请填写所有必填项"
- **当前状态**: 仅 toast 提示成功，1.5s 后跳转 `/dashboard`
- **TODO (Phase 2)**: 对接 `POST /api/projects` 真实创建

**前端文件**: `frontend/src/pages/CreatePage.jsx` (118行)

│  │  § 📍 里程碑                   │  └─────────────────┘│
│  │  § 📋 项目动态 (activities时)  │                     │
│  │  § 团队成员                     │                     │
│  └──────────────────────────────────┘                     │
└──────────────────────────────────────────────────────────┘
```

**数据加载**:
```js
useEffect(() => {
  getProjectDetail(id).then(data => { setDetail(data); setLoading(false) })
}, [id])
```

**字段映射兼容** (处理前后端字段不一致):
```js
p.description  || p.desc                    // 描述
p.authorName  || p.author                   // 发起人
c.authorName  || c.author                   // 评论作者
c.content     || c.text                    // 评论内容
f.uploaderName|| f.uploader                 // 文件上传者
```

**条件渲染规则**:

| 区块 | 条件 |
|------|------|
| 贡献者排行 | `p.status === 'done' && p.contributors` |
| 交付物 | `p.status === 'done' && p.deliverables` |
| 项目成就 (sidebar) | `p.status === 'done'` |
| Fork 按钮 | `p.status === 'done'` (否则显示收藏) |
| 加入按钮 | `p.status !== 'done'` |

**子组件 — TaskBoard**:
- 三列看板: `待认领(open)` / `进行中(progress)` / `已完成(done)`
- TaskCard: 任务名 + 负责人头像 + XP奖励 + [认领此任务]按钮
- 数据源: `milestones[].tasks[]` (扁平化后按 status 分组)

**子组件 — CollabSpace**:
- Tab 切换: 💬讨论 / 📂文件 / 📋项目日志
- 讨论 tab: 评论列表 + 底部输入框(avatar + textarea + 发送按钮)
- 文件 tab: 文件列表(图标+名称+上传者+时间+下载箭头)
- 日志 tab: activity-feed

**前端文件**: `frontend/src/pages/DetailPage.jsx` (406行)

### 4.3 HomePage 项目浏览区 (`#projects`)

**位置**: HomePage 的第 4 个 section

**筛选工具栏** (`.search-bar`):
```
┌──────────────────────────────────────────────────────┐
│ 🔍 [搜索项目名称、描述、角色…            ]            │
│                                                      │
│ [全部] [📱App] [🌐Web] [🎮Game] [🤖AI] [🔧Tool] | 全部 [招募中] [进行中] [已完成] │
│                                         找到 N 个项目  │
└──────────────────────────────────────────────────────┘
```

**筛选维度**:
- **分类 chip** (6个): `all/app/web/game/ai/tool` → 过滤 `(item.project || item).category`
- **状态 chip** (4个): `all/open/progress/done` → 过滤 `status`
- **搜索框**: 匹配 `标题 OR 描述 OR 角色名` (大小写不敏感)

**项目卡片网格** (`.projects-grid`): 3 列 (响应式: ≤1024px→2列, ≤768px→1列)
- 无结果 → 显示 `.no-results` (🔍 图标 + "没有找到匹配的项目")

**前端文件**: `frontend/src/pages/HomePage.jsx` (lines 86-112)

## 7. 项目状态流转

```
  open (招募中) ──→ progress (进行中) ──→ done (已完成)
     │                  │                      │
     │                  │                      │
  可编辑基本信息      可推进里程碑          锁定 + 生成
  可接受申请         可认领/完成任务       贡献者排行
  可加入成员                                交付物展示
```

| 状态 | 中文 | CSS Class | 颜色方案 |
|------|------|-----------|----------|
| `open` | 招募中 | `.tag-open` | 红 `rgba(212,33,61,0.1)` |
| `progress` | 进行中 | `.tag-progress` | 金 `rgba(212,168,67,0.15)` |
| `done` | 已完成 | `.tag-done` | 绿 `rgba(34,139,34,0.1)` |

**常量定义** (`constants.js`):
```js
STATUS_MAP    = { open: '招募中', progress: '进行中', done: '已完成' }
TAG_CLASS     = { open: 'tag-open', progress: 'tag-progress', done: 'tag-done' }
STATUS_COLORS = { done: {bg:'...',text:'var(--success)'}, progress:{...}, open:{...} }
```

## 8. 项目分类体系

| key | 显示名 | Emoji | 说明 |
|-----|--------|-------|------|
| `app` | 移动应用 | 📱 | iOS/Android/App |
| `web` | Web 应用 | 🌐 | 网站/WebApp/SaaS |
| `game` | 游戏 | 🎮 | 独立游戏/手游 |
| `tool` | 工具 | 🔧 | 开发工具/效率工具 |
| `ai` | AI / ML | 🤖 | 机器学习/AI 应用 |
| `other` | 其他 | — | 不属于以上类别 |

**项目 emoji 映射** (按 id): `'🍳','🧠','🔄','📋','👴','🎮'` → fallback `'📋'`
